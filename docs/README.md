## Contributing to Documentation

When making changes:
1. Update the relevant documentation file
2. Update this README if adding new documentation
3. Ensure examples are tested and accurate
4. Update version numbers if applicable
5. Add date of last update at the top of files

## Documentation Style

- Use clear, concise language
- Provide code examples where applicable
- Include expected output/error messages
- Use Markdown formatting consistently
- Link between related documents
- Keep documentation up to date with code changes

## Support

For documentation issues or questions:
- Open an issue on GitHub
- Tag with `documentation`
- Specify which document needs attention
- Provide suggestions for improvement
